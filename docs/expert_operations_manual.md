# 전문가 운영 매뉴얼 (초안)

## 1. 조직 구조 및 역할
- **운영팀 관리자**: 전문가 선발, 계약 관리, SLA 모니터링, Escalation.
- **전문가 패널**: 견적 분석, 코멘트 작성, 체크리스트 업데이트.
- **QA 담당자**: 리포트 품질 검토, 수정 요청 관리.

## 2. 전문가 선발·계약 절차
1. 지원/추천 접수 → 서류 심사(경력 20년 이상, 관련 자격) → 인터뷰.
2. 비밀유지/용역 계약 체결(기밀 유지, 책임 범위, 재사용 금지 조항).
3. 필수 서류: 신분증, 경력 증빙, 계좌 정보, 세금 신고용 서류.

## 3. 온보딩 프로세스
- 계정 생성 및 MFA 설정.
- 작업 도구 교육(전문가 콘솔, AI 시각화 해석 가이드, 체크리스트 사용법).
- SLA 규정, 저가 경고 안내, 계약금/중도금 검토 항목 교육.
- 시험 분석 2건 수행 → QA 통과 후 정식 투입.

## 4. 일일 운영 플로우
1. 하루 시작 전 SLA 대시보드 확인, 당일 할당량 파악.
2. 콘솔에서 작업 수락 → AI 요약 검토 → 항목별 체크/코멘트 작성.
3. 계약금/중도금/잔금 비율 확인, 저가 견적 사유 입력.
4. QA 제출 → 수정 요청 시 4시간 이내 대응.

## 5. QA 및 피드백
- QA 체크리스트 항목: 위험 플래그 적절성, 코멘트 명확성, 면허 확인 여부.
- 재검토 요청 시 “수정 사유/예시” 제공, 최대 2회.
- 주간 리뷰 미팅: 케이스 공유, 가이드 업데이트, 모델 개선 피드백 수집.

## 6. SLA 및 Escalation
- 기본 응답: Express 12h 이내 초안, Urgent 2h 이내 착수.
- 지연 예상 시 즉시 운영팀에 알림 → 대체 전문가 배정.
- 야간/주말 파견 근무자 리스트 관리(온콜 스케줄).

## 7. 성과 및 보상
- 측정 지표: 처리 건수, SLA 준수율, QA 재검토 비율, 고객 만족도.
- 월간 피드백: 상위 전문가 사례 공유, 개선 영역 조언.
- 보상 체계: SLA 준수·품질 기준 충족 시 성과 인센티브.

## 8. 문서화 및 업데이트
- 모든 변경 사항은 `docs/expert_operations_manual.md` 업데이트.
- 신규 가이드/템플릿은 콘솔 내 링크와 슬랙 채널에 공지.
- 연 1회 이상 정책 리뷰 및 개정.
