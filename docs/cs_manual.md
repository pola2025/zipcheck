# 고객 지원 · CS 매뉴얼

## 1. 조직 구성
- **CS 리드**: 서비스 정책 관리, 이슈 Escalation, 보고서 작성
- **1차 대응 상담사**: 일반 문의, 결제/계정 이슈 처리
- **전문 상담사**: 리포트 이의 제기, SLA 지연/보상 상담

## 2. 문의 채널
- 이메일: support@zipcheck.co.kr
- 전화: 02-000-0000 (평일 09:00~18:00)
- 헬프센터(FAQ + 티켓 시스템)

## 3. 문의 분류
| 카테고리 | 예시 | SLA |
| --- | --- | --- |
| 계정/로그인 | 비밀번호 재설정, OTP 문제 | 4시간 내 1차 답변 |
| 결제/영수증 | 결제 실패, 영수증 발급, 환불 문의 | 4시간 내 처리 |
| 리포트 이의 제기 | 분석 내용 재검토 요청 | 24시간 내 접수, 72시간 내 결과 |
| SLA 지연/보상 | 지연 알림, 보상 확인 | 2시간 내 안내 |
| 기타/제휴 | 제휴 문의, 기능 제안 | 24시간 내 응답 |

## 4. 처리 플로우
1. 티켓 접수 → 분류 태그 지정 → 담당자 배정
2. 필요 시 전문가/운영팀 협의 → 결과 회신
3. SLA 지연/보상 케이스: `sla_operations_guide` 절차 준수
4. 티켓 종료 후 만족도 설문 발송 (선택)

## 5. FAQ 구조
- 가입/로그인, 견적 업로드, 결제, 리포트 확인, 환불 불가 안내, SLA 지연, 저가 경고
- 각 문항은 캡처/영상 포함, 정기 업데이트

## 6. Escalation
- Level 1: 상담사 → 해결 불가 시 Level 2로 이관
- Level 2: 전문 상담/운영팀 → 기술/법무 이슈는 Level 3로 Escalation
- Level 3: CTO/법무팀 → 정책 변경 또는 긴급 대응

## 7. 품질 관리
- 티켓 처리 SLA 준수율, 고객 만족도(NPS/CSAT) 지표 추적
- 주간 피드백 회의, 우수/미흡 사례 공유
- 스크립트/FAQ 분기별 리뷰
