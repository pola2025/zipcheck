# SLA 운영 가이드 (초안)

## 1. SLA 정의
| 티어 | 약속 처리 시간 | 운영 시간 | 기본 요금 | 비고 |
| --- | --- | --- | --- | --- |
| Standard | 48시간 | 평일 09:00~18:00 | 30,000원 | 기본 서비스 |
| Express | 24시간 | 평일 09:00~18:00 | 45,000원 | +50% |
| Urgent | 3시간 | 평일 09:00~18:00 | 60,000원 | +100% |
| Late-Night Urgent | 익일 12시 전 | 평일 21:00~24:00 접수 | 120,000원 | +300% |
| Holiday Urgent | 3시간 | 주말/공휴일 09:00~18:00 | 120,000원 | +300% |

## 2. 프로세스
1. 결제 완료 → SLA 타이머 시작, 고객 및 운영팀 알림
2. 워커 배정: Urgent/Late/Holiday 우선 → Express → Standard
3. 진행 단계: 접수 → AI 분석 → 전문가 리뷰 → QA → 발송
4. 발송 후 자동 알림 + PDF 링크 제공

## 3. 모니터링 지표
- 티어별 처리시간(평균/중간값)
- SLA 준수율(지연 건수)
- 보상/환불 발생 건수
- 저가 경고 후 Escalation 건수

## 4. Escalation 규칙
- 남은 시간이 30분 미만인 Urgent/Holiday 작업: 온콜 담당자 호출
- 전문가는 지연 예상 즉시 Slack #sla-alert 채널에 공유 → 운영팀 재배정
- 시스템 장애 발생 시 Runbook(장애 유형별 대응) 참고

## 5. 보상 정책
- SLA 초과 시 자동 쿠폰 발행: Standard/Express 50% 할인 쿠폰, Urgent 이상 100% 환불
- 운영팀은 보상 처리 내역을 `payments`와 `operational_metrics`에 기록
- 반복 지연 원인 분석 리포트 주간 공유

## 6. 기록 및 보고
- 일일 SLA 리포트: 전체 티어 처리량, 지연 건, 보상 처리
- 주간 회의: 반복 지연 요인, 전문가 리소스 조정, 시스템 개선 사항
- 월간 리포트: SLA 추세, 고객 만족도, 개선 계획
